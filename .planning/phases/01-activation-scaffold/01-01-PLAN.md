  <files>users/michael/windows/default.nix</files>
  <action>
Create `users/michael/windows/default.nix` — the core scaffold module. This file does three things:

1. **Declare the `windows.configFiles` option** (unconditionally, outside `mkIf`):
   - Type: `lib.types.attrsOf lib.types.path`
   - Default: `{ }` (empty attrset — no files registered in Phase 1)
   - Description: Keys are relative paths under `$windowsHomePath/.config/`, values are Nix store paths

2. **Guard all config on `isWSL`** using `lib.mkIf`:
   - Access `osConfig.hostSpec.isWSL or false` (with `osConfig ? { }` default in function args for safety)
   - Wrap only the `config` section in `lib.mkIf isWSL { ... }`
   - Options MUST remain outside the guard so non-WSL hosts can still evaluate the type

3. **Implement `home.activation.syncWindowsConfigs`** hook inside the guarded config:
   - Use `lib.hm.dag.entryAfter [ "writeBoundary" ]` (runs after HM writes all managed files)
   - Define `windowsHomePath = "/mnt/c/Users/michael"` as a let-binding
   - Mount guard: `if [ ! -d "${windowsHomePath}" ]; then echo "[windows-sync] ... not accessible — skipping"; exit 0; fi`
   - Use `lib.concatStringsSep "\n" (lib.mapAttrsToList ...)` to generate per-file copy commands at Nix eval time
   - For each entry: `mkdir -p` the parent directory, then copy with `cp` for files or `cp -r` for directories (detect with `[ -d ... ]`)
   - Prefix all write operations with `$DRY_RUN_CMD` (matching darwin sops pattern)
   - Log each file: `echo "[windows-sync] ${relPath} -> synced"` on success
   - Track failures with a `_failed` variable; report summary at end but do not abort on individual failures
   - Set 644 permissions on copied files with `chmod 644`

Module shape (follow this structure exactly):
```nix
{
  lib,
  osConfig ? { },
  config,
  ...
}:
let
  isWSL = osConfig.hostSpec.isWSL or false;
  cfg = config.windows;
  windowsHomePath = "/mnt/c/Users/michael";
in
{
  options.windows.configFiles = lib.mkOption { ... };

  config = lib.mkIf isWSL {
    home.activation.syncWindowsConfigs = lib.hm.dag.entryAfter [ "writeBoundary" ] ''
      ...
    '';
  };
}
```

- Use `entryBefore` (derivation paths must exist first, so after writeBoundary)
- Create per-app subdirectories yet (those arrive in Phases 2-4)
- Add test/placeholder files (per user decision: scaffold is empty until Phase 2)
